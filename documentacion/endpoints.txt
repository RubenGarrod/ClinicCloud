/search (definido en search.py):

Método HTTP: POST
Propósito: Realizar búsquedas de documentos utilizando una consulta en lenguaje natural.
Razón para usar POST: La búsqueda requiere enviar un cuerpo JSON con la consulta y parámetros adicionales (como limit y offset), lo cual es más adecuado para POST.



/documents (definido en otro archivo):

Método HTTP: GET
Propósito: Obtener una lista de documentos almacenados en la base de datos.
Razón para usar GET: Este endpoint simplemente recupera datos sin necesidad de enviar un cuerpo en la solicitud.



/categories (definido en otro archivo):

Método HTTP: GET
Propósito: Obtener una lista de categorías disponibles o una categoría específica.
Razón para usar GET: Similar a /documents, este endpoint solo recupera datos.
